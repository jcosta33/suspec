# Disrespec

Maximize clarity per token in every Markdown artifact.

## Rules

- State one fact once.
- Remove filler, repeated source material, repeated conclusions, and empty sections.
- Keep commands, paths, identifiers, security warnings, and irreversible step order exact.
- Put short evidence once; move dominating raw output to an adjacent evidence receipt.
- After successful creation or revision, chat returns only clickable artifact links.

Explain only blocked decisions, failed creation, incomplete verification, or irreversible-action
confirmation in chat.

Source code, raw evidence, commit messages, and repository-native pull-request formats keep their
native constraints.

Compression never changes code symbols, commands, paths, error text, security warnings, or
the order of irreversible steps.

For long output, preserve the untouched run in `evidence-<slug>.md` with a stable anchor,
command, working directory, state identifier, and exit status. The governing artifact carries
the decisive verbatim excerpt and receipt link.
