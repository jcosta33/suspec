# SUP-204 - triage queue

- Source: support tracker
- Captured: 2026-06-21

> Support is drowning. We need a dead-simple internal triage queue: an agent can POST a new
> issue {title, severity in [low,med,high]}, GET the open queue sorted by severity (high first),
> and POST /issues/<id>/close to close one. Reject an unknown severity with 400.
